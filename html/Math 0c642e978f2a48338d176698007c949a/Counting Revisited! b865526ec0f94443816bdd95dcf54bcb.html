We have now modulo by our side! Let’s deal with big guys, shall we?

Without the modulo operation, we don’t have a feasible option in C++.

Using strings to represent unbounded integers and implementing arithmetic operations on strings is always an option, but this would be computationally expensive as numbers get enormous.

So, we just use remainders!

Factorials

n!=1⋅2⋯nn! = 1 \cdot 2 \cdots nn!=1⋅2⋯n﻿ is just the product of nnn﻿ integers. We can iteratively calculate n!n!n!﻿ by multiplying these integers one by one.

Implementation
// https://ideone.com/OLseOi
#include <bits/stdc++.h>
using namespace std;

const int mod = 1e9 + 7;

int mul(int a, int b) {
    return 1ll * a * b % mod;
}

int factorial(int n) {
    int res = 1;
    for (int i = 1; i <= n; i++) {
        res = mul(res, i);
    }
    return res;
}

int factorial_recursive(int n) {
    if (!n)
        return 1;
    return mul(n, factorial_recursive(n - 1));
}

void print_factorials(int bound) {
    int n_examples = 10;
    for (int i = 0; i <= n_examples; i++) {
        int n = rand() % (bound + 1);
        cout << n << "!:\n";
        cout << "\t" << factorial(n) << " (iterative)\n";
        cout << "\t" << factorial_recursive(n) << " (recursive)\n";
    }
}

int main() {
    int bound = 1e7;
    print_factorials(bound);
}

        res %= MOD

def factorial_recursive(n: int) -> int:
    if not n:
        return 1
    return n * factorial_recursive(n - 1) % MOD

Binomial Coefficients

See it as a fraction pq\frac{p}{q}qp​﻿. We can use qm−1≡1(modm)q^{m-1} \equiv 1 \pmod mqm−1≡1(modm)﻿ here, where qqq﻿ is the denominator (n−k)!⋅k!(n-k)! \cdot k!(n−k)!⋅k!﻿ which is coprime with m=109+7m = 10^9 + 7m=109+7﻿ for k≤n≤107k \le n \le 10^7k≤n≤107﻿, because both n−kn - kn−k﻿ and kkk﻿ are less than mmm﻿. So, we are multiplying pq\frac{p}{q}qp​﻿ by qm−1q^{m-1}qm−1﻿ to just calculate p⋅qm−2p \cdot q^{m-2}p⋅qm−2﻿. Recall that we can use binary exponentiation here.

There are a few important tricks used for one of the best implementations:

  • We often precalculate i!i!i!﻿’s for i∈[1,n]i \in [1, n]i∈[1,n]﻿ (store in an array) to then calculate a binomial coefficient in O(log⁡m)\mathcal{O}(\log m)O(logm)﻿ time whenever needed (because of binary exponentiation). Because, problems often require lots of binomial coefficients with different (n,k)(n, k)(n,k)﻿’s. I strongly recommend you todo precalculation like so. It won’t harm, and is pretty advantageous.
  • Note that q=(n−k)!⋅k!q = (n-k)! \cdot k!q=(n−k)!⋅k!﻿. To calculate qm−2q^{m-2}qm−2﻿, we can either calculate (q mod m)m−2(q \bmod m)^{m-2}(qmodm)m−2﻿ or [((n−k)!m−2 mod m)⋅(k!m−2 mod m)] mod m\left[\left((n-k)!^{m-2} \bmod m \right) \cdot \left(k!^{m-2} \bmod m \right) \right] \bmod m[((n−k)!m−2modm)⋅(k!m−2modm)]modm﻿. This means we can choose to separately calculate the factorials’ powers to m−2m-2m−2﻿, then multiply them modulo mmm﻿. So, i!m−2i!^{m-2}i!m−2﻿’s can be precalculated in O(nlog⁡m)\mathcal{O}(n \log m)O(nlogm)﻿. This optimizes the precalculation explained previously to O(1)\mathcal{O}(1)O(1)﻿, because no binary exponentiation is needed after this extra precalculation.
    i!m−2i!^{m-2}i!m−2﻿ is also called inverse factorial!
  • Inverse factorials can even be precalculated in O(n+log⁡m)≈O(n)\mathcal{O}(n + \log m) \approx \mathcal{O}(n)O(n+logm)≈O(n)﻿, considering the following property:

    It actually represents i+1(i+1)!=1i!\dfrac{i+1}{(i+1)!} = \dfrac{1}{i!}(i+1)!i+1​=i!1​﻿.

    So, we can start by calculating n!m−2n!^{m-2}n!m−2﻿ in O(log⁡m)\mathcal{O}(\log m)O(logm)﻿ using the already calculated n!n!n!﻿, then calculate i!i!i!﻿’s from n−1!n-1!n−1!﻿ to 0!0!0!﻿ in O(1)\mathcal{O}(1)O(1)﻿ per each.

As a result, factorials and their inverses can be precalculated in O(n)\mathcal{O}(n)O(n)﻿, then a single binomial coefficient can be calculated in O(1)\mathcal{O}(1)O(1)﻿.

Implementation
// https://ideone.com/HFQx8j
#include <bits/stdc++.h>
using namespace std;

const int N = 1e6 + 5;
const int mod = 1e9 + 7;
int f[N], inv_f[N];

int mul(int a, int b) {
    return 1ll * a * b % mod;
}

// base^(mod - 2)
int invexp(int base) {
    int result = 1;
    int exponent = mod - 2;
    while (exponent) {
        if (exponent & 1)
            result = mul(result, base);
        exponent >>= 1;
        base = mul(base, base);
    }
    return result;
}

void precalculate() {
    f[0] = 1;
    for (int i = 1; i < N; i++) {
        f[i] = mul(f[i - 1], i);
    }

    inv_f[N - 1] = invexp(f[N - 1]);
    for (int i = N - 2; i >= 0; i--) {
        inv_f[i] = mul(i + 1, inv_f[i + 1]);
    }
}

void print_binomial_coefficients(int upper_bound) {
    // An example so-called "lambda" function, similar to those in Python.
    // I used it to just let you know about them.
    auto C = [&](int n, int k) {
        if (n < k)
            return 0;
        return mul(f[n], mul(inv_f[n - k], inv_f[k]));
    };

    int n_examples = 10;
    for (int i = 0; i < n_examples; i++) {
        int n = rand() % (N + 1), k = rand() % (N + 1);
        cout << "C(" << n << ", " << k << ") mod (1e9 + 7): " << C(n, k) << "\n";
    }
}

int main() {
    // To make program generate different sequences of random integers for each run.
    srand(time(0));
    precalculate();
    int upper_bound = 1e6;
    print_binomial_coefficients(upper_bound);
}

Actual Result

These precalculation tricks are only feasible for small nnn﻿’s bounded by some number like 10710^7107﻿ (think about the memory usage also).

We can do something else without precalculation for n≤1018,∣n−k∣≤107n \le 10^{18}, |n - k| \le 10^7n≤1018,∣n−k∣≤107﻿. Notice how we can simplify the formula as follows when we divide n!n!n!﻿ by (n−k)!(n - k)!(n−k)!﻿ (or k!k!k!﻿ if it’s larger):

This time, it’s straightforward — calculate n⋅(n−1)⋯(n−k+1)⋅k!m−2n \cdot (n-1) \cdots (n-k+1) \cdot k!^{m-2}n⋅(n−1)⋯(n−k+1)⋅k!m−2﻿.

Implementation
// https://ideone.com/5V5FFT
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int mod = 1e9 + 7;

int mul(int a, int b) {
    return 1ll * a * b % mod;
}

// base^(mod - 2)
int invexp(int base) {
    int result = 1;
    int exponent = mod - 2;
    while (exponent) {
        if (exponent & 1)
            result = mul(result, base);
        exponent >>= 1;
        base = mul(base, base);
    }
    return result;
}

int C(ll n, ll k) {
    if (n < k)
        return 0;
    // n * (n-1) * (n-2) * ... * (n-k+1) / k!
    k = min(k, n - k); // to guarantee k is small
    int numerator = 1, denominator = 1;
    for (ll i = n; i > n - k; i--) {
        numerator = mul(numerator, i % mod);
        denominator = mul(denominator, (i - (n - k)) % mod);
    }
    return mul(numerator, invexp(denominator));
}

void print_binomial_coefficients(int upper_bound) {

    int n_examples = 10;
    for (int i = 0; i < n_examples; i++) {
        ll n = 1ll * rand() * rand();
        bool k_is_big = rand() % 2;
        ll k = rand() % (upper_bound + 1);
        if (k_is_big)
            k = n - rand() % (upper_bound + 1);
        cout << "C(" << n << ", " << k << ") mod (1e9 + 7): " << C(n, k) << "\n";
    }
}

int main() {
    // To make program generate different sequences of random integers for each run.
    srand(time(0));
    int upper_bound = 1e6;
    print_binomial_coefficients(upper_bound);
}
